## 初次使用

1. 下载 `index.html` 文件
2. 双击文件，或将其拖入浏览器窗口
3. 应用立即可用，无需任何安装步骤

**推荐浏览器：** Chrome 90+、Firefox 88+、Safari 14+、Edge 90+

> **提示：** 你的数据自动保存在浏览器本地存储中，关闭并重新打开后数据仍在。

## 界面介绍

FluxSQL 分为三个主要区域：

```
┌─────────────────────────────────────────────────────┐
│  顶部工具栏：方言切换 | 导入 | 导出 | 新建表 | AI    │
├──────────┬──────────────────────────────────────────┤
│          │   字段编辑区                              │
│  表列表  │   - 字段名                               │
│  (左侧栏)│   - 数据类型                              │
│          │   - 约束设置                              │
│          │   - 注释                                  │
│          ├──────────────────────────────────────────┤
│          │   AI 对话面板（可折叠）                   │
└──────────┴──────────────────────────────────────────┘
```

### 顶部工具栏

| 按钮 | 功能 |
|------|------|
| **Oracle/MySQL/PostgreSQL/SQL Server** | 切换 SQL 方言 |
| **+ 新建表** | 手动创建新表 |
| **导入** | 导入 SQL 或 JSON 文件 |
| **导出 SQL** | 导出当前所有表的 DDL |
| **导出 JSON** | 导出表结构为 JSON 格式 |
| **✨ AI 建表** | 打开 AI 对话，用自然语言建表 |

## 创建数据库表

### 方法一：手动创建

1. 点击顶部 **「+ 新建表」** 按钮
2. 在弹出框中输入表名和可选的表注释
3. 点击确认，进入字段编辑界面
4. 点击 **「+ 添加字段」** 开始添加字段

### 方法二：使用 AI 创建（推荐）

见下方 [AI 智能建表](#ai-智能建表) 章节。

## AI 智能建表

### 配置 API 密钥

1. 点击顶部 **「⚙ 设置」** 或 **「AI 配置」**
2. 选择你的 AI 服务商（推荐 DeepSeek，价格实惠）
3. 输入 API Key 和对应的模型名称
4. 点击保存

> API 密钥保存在浏览器本地，不会上传到任何服务器。

详细配置见 [api-config.md](./api-config.md)

### 使用 AI 建表

1. 点击底部 **「✨ AI 建表」** 展开 AI 对话面板
2. 在输入框中描述你的表结构，例如：
   ```
   创建一个用户表，包含ID、用户名、邮箱、手机号、密码、头像URL、创建时间、最后登录时间，邮箱和手机号需要唯一约束
   ```
3. 点击发送按钮（或按 `Ctrl+Enter`）
4. AI 会分析描述并生成字段方案
5. 确认无误后点击 **「执行」** 即可应用到表结构

### AI 快捷模板

点击输入框上方的快捷模板按钮，快速生成常见表：
- 👤 用户表
- 📦 订单表
- 🛍️ 商品表
- 📝 文章表
- 📁 文件表

### AI 模式说明

| 模式 | 适用场景 |
|------|---------|
| **建表模式** | 从零开始描述创建新表 |
| **修改模式** | 基于现有表描述修改字段 |
| **分析模式** | 对现有表提问、优化建议 |

## 字段编辑

### 添加字段

在字段列表下方点击 **「+ 添加字段」** 按钮。

### 字段属性说明

| 属性 | 说明 |
|------|------|
| **字段名** | 列名，遵循所选数据库命名规范 |
| **数据类型** | 从下拉菜单选择，会根据当前方言自动过滤 |
| **长度/精度** | 适用于 VARCHAR、DECIMAL 等类型 |
| **PK** | 设为主键（自动添加 NOT NULL） |
| **NN** | NOT NULL 约束 |
| **UQ** | UNIQUE 约束 |
| **AI** | 自增（AUTO_INCREMENT / SERIAL / IDENTITY） |
| **默认值** | 列的默认值，支持函数（如 CURRENT_TIMESTAMP） |
| **注释** | 字段说明，用于生成 COMMENT 子句 |

### 拖拽排序

按住字段行左侧的拖拽图标（⠿），可以上下拖动调整字段顺序。

### 删除字段

点击字段行右侧的 **「×」** 按钮删除该字段。

## 数据库方言切换

在顶部方言选择器中点击对应数据库名称：

- **Oracle** — 生成 Oracle 风格 DDL（NUMBER、VARCHAR2、SEQUENCE 等）
- **MySQL** — 生成 MySQL 风格 DDL（AUTO_INCREMENT、ENGINE=InnoDB 等）
- **PostgreSQL** — 生成 PostgreSQL 风格 DDL（SERIAL、TEXT 等）
- **SQL Server** — 生成 SQL Server 风格 DDL（IDENTITY、NVARCHAR 等）

切换时，SQL 预览区域会立即更新。不兼容的字段类型会有黄色警告提示。

## 导入 SQL

1. 点击顶部 **「导入」** 按钮
2. 选择导入方式：
   - 粘贴 SQL 文本
   - 上传 `.sql` 文件
   - 上传 `.json` 文件（FluxSQL 格式）
3. 点击确认，FluxSQL 会解析并导入表结构

> **支持的 SQL 格式：** CREATE TABLE 语句，支持大多数主流数据库的 DDL 语法。

---

## 导出 SQL / JSON

### 导出 SQL

1. 点击 **「导出 SQL」** 按钮
2. 复制到剪贴板，或下载为 `.sql` 文件
3. 输出为标准 DDL 语句，适合直接在数据库中执行

### 导出 JSON

1. 点击 **「导出 JSON」** 按钮
2. 输出 FluxSQL 内部格式的 JSON
3. 可用于备份或在不同设备间迁移工作

## 常见问题

**Q: AI 建表不响应怎么办？**  
A: 检查 API Key 是否正确配置，以及是否有网络连接（AI 功能需要访问 API 服务器）。

**Q: 关闭浏览器后数据还在吗？**  
A: 是的！数据保存在浏览器的 localStorage 中，只要不清除浏览器数据就不会丢失。建议定期导出 JSON 备份。

**Q: 能在手机上使用吗？**  
A: 可以，界面做了基本的响应式适配，但推荐使用桌面浏览器以获得最佳体验。

**Q: 导入的 SQL 报错怎么办？**  
A: 请确保 SQL 是标准的 `CREATE TABLE` 语句。高度定制化的存储过程或数据库特定语法可能无法解析。

**Q: 支持哪些 AI 服务商？**  
A: 支持所有兼容 OpenAI API 格式的服务商，包括 OpenAI、DeepSeek、Kimi、智谱 AI 等。
